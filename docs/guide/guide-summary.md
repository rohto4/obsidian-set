# Guide Summary

`docs/guide/` は、採用済み、または採用寄りの恒久運用ガイド、判断基準、ルール補足を置く場所。

| File | 1行説明 |
|---|---|
| `ai-knowledge-taxonomy.md` | AI 関連知識を整理する初期カテゴリと freshness 方針。 |
| `ai-session-summary.md` | AI セッション要約を保存する場所、保存対象、テンプレートのガイド。 |
| `dev-knowledge-taxonomy.md` | 開発ノウハウを整理する初期カテゴリと note type の対応方針。 |
| `guide-summary.md` | `docs/guide/` 内のファイル一覧と1行説明。 |
| `inbox-policy.md` | 未整理情報を一時的に置く inbox の使い方、昇格先、処理ルール。 |
| `knowledge-interface-catalog.md` | vault 知識へ AI agent が問い合わせるための候補インターフェース一覧。 |
| `query-protocol.md` | AI に vault 内知識を問い合わせる時の読み込み順、出力形式、根拠ルール。 |
| `source-backed-notes.md` | 根拠資料に基づく note の保存先、source class、テンプレート。 |
| `sql-queryable-vault.md` | Dataview、SQLite、DuckDB などで vault を検索可能にする方式整理。 |
| `sync-strategy.md` | Obsidian Sync と GitHub repo の併用方針、衝突ポリシー案。 |
| `vault-operation.md` | 本運用 vault の場所、構成、正本、重複管理、検索方針。 |
| `web-clipper.md` | Obsidian Web Clipper の使いどころ、運用想定、初期テンプレート案。 |
