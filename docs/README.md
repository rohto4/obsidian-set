# docs Structure

このディレクトリは、`obsidian-set` の運用ルール、検討資料、仕様、作業記録を置く場所。

| Path | 役割 |
|---|---|
| `docs/README.md` | `docs/` 全体の構成一覧。docs 配下の構成が変わったら更新する。 |
| `docs/ecc-obsidian-setup.md` | ECC をこの PJ にどう取り込んだかの導入メモ。 |
| `docs/memo.md` | 初回依頼などの生メモ。恒久ルールの正本にはしない。 |
| `docs/guide/` | 採用済み、または採用寄りの恒久運用ガイド、判断基準、ルール補足。 |
| `docs/guide/guide-summary.md` | `docs/guide/` 内のファイル一覧と1行説明。 |
| `docs/condi-ref/` | 条件付き参考資料、実現可能性確認、候補比較、未採用の選択肢。 |
| `docs/condi-ref/condi-ref-summary.md` | `docs/condi-ref/` 内のファイル一覧と1行説明。 |
| `docs/spec/` | 実装対象として確定した仕様、要件、設計前提。 |
| `docs/spec/spec-summary.md` | `docs/spec/` 内のファイル一覧と1行説明。 |
| `docs/imp/` | 実装メモ、作業記録、進捗、引き継ぎ。 |
| `docs/Z_trash/` | 退避した旧資料や現時点で正本ではない資料。 |
